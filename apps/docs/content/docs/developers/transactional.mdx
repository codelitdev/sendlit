---
title: Transactional email
description: Send a single email through the REST API.
---

Use the transactional send endpoint for a single, API-triggered message. This is appropriate for application events such as receipts, invitations, password resets, and notifications.

Transactional sends are recorded in the dashboard’s **Transactional log**, where you can inspect status and provider feedback. They do not enroll a contact in a sequence and do not behave like a broadcast audience send.

See the API reference for the request schema, template variables, sender fields, and response identifiers.
